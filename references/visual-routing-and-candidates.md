# 视觉路由与多候选方向

用于在生成星禾视觉资产前判断“该画什么、画几种方向、是否适合真实生图”。本文件借鉴视觉路由思想，但保持星禾 IP、白底蜡笔、中文可读和平台规则。

## 视觉路由

| 路由 | 适合内容 | 输出方式 |
|---|---|---|
| `xinghe-article` | 文章正文中的概念、流程、判断、复盘、场景瞬间 | 星禾正文配图候选方向 |
| `platform-cover` | 公众号封面、小红书封面、文章头图、社媒首图 | 平台封面候选方向 |
| `emotion-anchor` | 情绪冲突、误区真相、理想现实反差 | 情绪锚点图候选方向 |
| `explanatory-diagram` | 抽象概念、轻量流程、因果链、机制解释 | 手绘解释图候选方向 |
| `comic-strip` | 因果、累积、转折、前后变化 | 2-4 格漫画候选方向 |
| `knowledge-card-pack` | 长文总结、小红书轮播、方法论拆卡 | 5-9 张知识卡片组 |
| `knowledge-card-single` | 单观点、单流程、单对比、单清单 | 单张知识卡片 |
| `infographic-poster` | 整篇流程、矩阵、地图或全局总览 | 默认最多 1 张信息图海报 |
| `diagram-fallback` | 精确技术拓扑、数据库依赖、权限流、超过 8 个强关系节点 | 不强行套星禾，输出结构图建议或 prompt-only |
| `prompt-only` | 用户明确不要生图、环境无法传人物基准图、缺少 API/key/endpoint | 只输出候选方向和完整 prompt |

判断规则：

- 如果核心价值来自星禾动作、隐喻和个人 IP 记忆点，走 `xinghe-article` 或 `platform-cover`。
- 如果核心价值来自轻量结构、因果解释、步骤或对比，优先走 `explanatory-diagram`、`knowledge-card-single` 或 `infographic-poster`，不是默认降级。
- 如果核心价值来自精确技术拓扑、数据库依赖、权限流或强关系节点过多，才走 `diagram-fallback`。
- 如果用户只要策略、评审、测试、优化 skill，走 `prompt-only`。

## 候选数量

| 场景 | 默认候选数 | 说明 |
|---|---:|---|
| 正文配图普通选点 | 2 | 一个偏概念隐喻，一个偏动作流程 |
| 正文配图重点选点 | 3 | 增加一个偏场景/情绪/记忆点方向 |
| 公众号封面 | 3 | 标题强表达、人物动作强表达、留白/品牌感强表达 |
| 小红书封面 | 3 | 大字标题强表达、底部星禾动作强表达、关键词强调强表达 |
| 知识卡片组 | 先定卡片数，再为关键卡给 2-3 个方向 | 默认 5-9 张卡，关键卡可多候选 |
| 多格漫画 | 1-2 | 先验证每格是否推进意义 |
| 信息图海报 | 1-2 | 默认最多真实生成 1 张 |
| 快速任务或省成本 | 1 | 用户明确要快、要少、只要最终图时使用 |

## 候选方向字段

每个候选方向必须写清：

- 候选编号：`A`、`B`、`C`
- 核心隐喻：这张图用什么低科技物件或动作承载主题
- 星禾动作：星禾正在亲手解决什么问题
- 构图：标题/人物/物件的位置关系
- 中文标注：最多 5-8 个短词
- 推荐参考图：人物基准图 + 1 张正文或封面参考图
- 适用原因：为什么这个方向值得生成
- 建议比例：3:4、4:3、1:1、2.35:1 或 16:9
- 生成文件名：如 `card-02-a.png`、`comic-01-a.png`、`poster-a.png`

## 输出规则

- 策略模式：每个配图点输出候选方向，不调用生图。
- prompt-only：每个候选方向输出独立 prompt。
- 真实生成：先确认生成哪个候选；如果用户要求多个候选，分别生成独立图片。
- 文件命名：正文候选用 `01-topic-a.png`、`01-topic-b.png`；封面候选用 `cover-a.png`、`cover-b.png`、`cover-c.png`。
- 不覆盖旧图；已有文件时换新序号或询问用户。

## Diagram fallback

遇到以下内容时不要硬画星禾：

- 多层系统架构图、技术依赖图、复杂流程图。
- 节点超过 8 个，且节点关系比人物动作更重要。
- 时序、状态机、数据库/服务拓扑、权限流。

输出方式：

- 说明“不建议做星禾 IP 图”的原因。
- 给出结构图 prompt、Mermaid/Excalidraw 方向，或建议拆成多张星禾图。
- 如果用户坚持星禾风格，只画一个“理解复杂系统的场景瞬间”，不要画完整架构。
