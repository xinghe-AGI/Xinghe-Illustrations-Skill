# 构图模式与原创规则

## 使用边界

本文件用于常规正文配图和文章锚点图的构图，不负责公众号文章封面、小红书笔记封面、平台首图、知识卡片组、多格漫画或信息图海报。

- 正文配图、文章插图、常规星禾锚点图：读取本文件。
- 解释图、知识卡片、多格漫画、信息图海报：优先读取 `visual-formats.md`。
- 微信公众号封面、小红书笔记封面、平台首图：读取 `cover-composition-patterns.md`。
- 人物形象一致性：始终以 `assets/examples/00-xinghe-ip-baseline.png` 为基准。
- 正文配图参考图：优先从 `assets/examples/01-14-*.png` 中选择。
- 封面参考图：优先从 `assets/examples/15-20-*.png` 中选择，不要用正文配图构图硬套封面。

## 基础结构类型

选择一种结构即可，不要混太多。

先按 `illustration-selection.md` 判断图型，再选结构类型。图型帮助理解内容，结构类型帮助落地画面；二者都不能写成画面左上角标题。

| 图型 | 推荐结构 |
|---|---|
| concept | 灵感生产小装置 / 运营白板 / 清醒判断 |
| process | 内容流水线 / 自动化小盒子 / 小分镜 |
| comparison | 运营白板 / 清醒判断 |
| data | 复盘回流 / 清醒判断 |
| scene | 内容流水线 / 小分镜 |
| metaphor | 灵感生产小装置 / 自动化小盒子 / 清醒判断 |
| handoff | 内容流水线 / 自动化小盒子 |
| review-loop | 复盘回流 / 内容流水线 |

### 灵感生产小装置

适合：选题灵感、创意启动、把想法变成内容。

画法：星禾举起、打开或倾倒灵感瓶，星星或小纸条流入一个低科技内容成型装置，输出内容卡片。橙色线表达主流向。只有这类灵感场景才默认使用举瓶动作。

### 内容流水线

适合：选题、生成、编辑、发布、复盘。

画法：左侧输入灵感或素材，中间星禾根据问题做具体操作：搬开卡住的素材箱、接住掉落内容、调整传送带、把内容卡放回轨道，右侧输出内容成果。

### 运营白板

适合：SOP、账号定位、内容策略、私域承接。

画法：一个不规则小白板或便签区，只放 3-5 个关键节点。星禾用马克笔圈出、擦掉、重排或连接最重要的一步，不要只是站着指板。白板周围可以加入夹子、胶带、侧边短提示、底部笔记本或桌面小物，让背景有写作现场感；这些辅助元素只承接阅读路径和检查点，不复制旧参考图。

### 白板写作台

适合：推理流程、解释图、方法步骤、知识卡片横版、文章正文里的“把问题讲清楚”场景。

画法：画面主体是一块白板或写作板，中部承载主流程或主结论，边缘用 2-4 个便签写前提、提醒、问题或小结，底部用桌面层放笔、胶带、卡片、打开的笔记本来减少无功能空白。星禾可以在左下或右侧贴卡、拉线、圈重点、递标签；如果结构是主角，人物可以缩小或只露手。

禁忌：不要做成课程 PPT；不要把边缘便签写成长段；不要把桌面小物当随机装饰。

### 自动化小盒子

适合：AI 工作流、自动发布、素材整理、日报、监控。

画法：一个低科技小盒子接收输入、吐出输出。星禾负责插线、按按钮、拧旋钮、贴标签或把重复任务卡塞进盒子。

### 内容日历

适合：发布节奏、选题排期、栏目规划。

画法：一个简化月历或 7 格小格子，星禾把灵感卡片放进日期格。不要画真实日历界面。

### 复盘回流

适合：数据反馈、内容优化、下一轮选题。

画法：一个轻量回环，输出卡片变成蓝色反馈纸条回到灵感池。星禾拉起回流线、夹住反馈纸条、用蓝色笔把反馈送回下一轮输入。

### 多形态内容

适合：一篇内容多端改写、矩阵账号、素材复用。

画法：星禾把同一个星星灵感压印、裁切、折叠或分装成几张不同形状的内容卡，例如短帖、长文、视频脚本。

### 清醒判断

适合：问题本质、策略判断、优先级、复盘结论。

画法：星禾用放大镜、小秤或红色圈注找出关键点。画面更安静，留白更多。

### 小分镜

适合：正文配图里的轻微前后变化、一个小转折、从混乱到清楚。

画法：2-3 个小场景，每格只表达一个动作。它是正文图里的轻分区，不等于 `comic-strip` 多格漫画。真正需要 2-4 格因果叙事时，按 `visual-formats.md` 的多格漫画规则处理。

### 工作区整理台

适合：CodexWorkspace、项目台账、rules、skills 清单、issue log、自动化台账、工作区系统和方法论沉淀。

画法：星禾把散落的对话卡、规则卡、项目卡、复盘卡整理进低科技文件盒、台账格或工作区抽屉。画面重点是“把临时对话沉淀成可复用系统”。

禁忌：不要画成正式软件界面、PPT 系统架构图、复杂仪表盘或真实 IDE 截图；不要把所有治理文件名都塞进画面。

## 原创隐喻生成法

每次都从当前文章重新发明隐喻，不能照搬旧图。

### 三步

1. 把抽象概念换成一个物理动作：捕捉、装瓶、接线、排期、回流、分发、称量、圈出、折叠。
2. 把系统结构换成一个低科技物件：灵感瓶、内容卡、小白板、日历格、自动化盒子、小秤、放大镜、工具盒、门把手。
3. 让星禾承担动作：不是站旁边，也不是每次举瓶，而是亲手搬、接、插线、按、拉、贴、排、称、圈、铺证据、搭桥、分发或回流。

### 可用物件池

- 透明灵感瓶、星形灵感、内容卡片、小白板、便签、月历格、工具盒、邮筒、门、按钮盒、细线、小标签、放大镜、小秤、蓝色反馈纸条。
- 用时只选 1-2 个，不要堆满。

### 星禾动作池

- 举、倒、抓、指、拉、接、贴、排、按、收、分发、回流、圈出、称量、整理、挂起。
- 动作要服务核心意思，不要为了热闹而热闹。

## 反复刻规则

不要默认打开或复刻 `assets/examples/`。这些图片用于校准星禾人物形象、留白、批注密度、手绘草图感和封面排版关系。

除非用户明确说“照这张 / 复刻这个构图 / 用这个案例改”，否则不要直接复用旧参考图构图或旧参考图物件。

同类主题也要换新隐喻。例如“内容承接”不一定画路线，可以画星禾把内容纸条接到门把手；“一鱼多吃”不一定画鱼，可以画星禾把一个灵感瓶倒出几种内容卡。
