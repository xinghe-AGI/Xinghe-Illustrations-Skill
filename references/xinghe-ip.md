# 星禾 IP

## 角色定义

星禾是中文正文配图里的固定个人 IP。她不是贴纸、头像装饰或可爱吉祥物，而是一个会把灵感抓住、把复杂问题看清楚、把内容和运营流程跑起来的创作陪跑者。

星禾的综合定位：

- 元气创造者：负责捕捉灵感，把想法变成内容。
- 运营小导师：负责把内容、工具、发布和复盘串成闭环。
- 清醒研究员：负责拆解问题、称量证据、圈出关键判断。

默认正文图、封面、情绪图和多格漫画要出现星禾。技术架构图、流程图和高密度知识卡片可以让星禾变小、只露手/半身/侧影，或完全不出现。星禾出现时必须承担画面的核心动作，不能只是站在旁边当装饰。

## 外形

- 年轻女性个人 IP。
- 黑色长发或微卷长发，发量自然；额前有轻薄空气刘海，发尾自然散开，线条有蜡笔颗粒感。
- 圆润大眼睛，眼神明亮、亲切、专注；五官简单，不追求写实肖像。
- 温柔浅笑，带一点真实生活照里的松弛和亲近感，不夸张卖萌。
- 白色宽松拉链连帽外套，帽子体积感明显；衣褶、拉链和袖口用少量蓝色或黑蓝色蜡笔线表现。
- 内搭固定为深蓝水手领上衣，领口有白色双线条和白色领结；下身固定为深蓝百褶裙。服饰要简洁、可识别，不要画成复杂 cosplay 或写实校服。
- 脸颊可有极少量暖橙色腮红，但不要大面积上色。
- 常见识别物：透明小玻璃灵感瓶、星星、内容卡片、马克笔、小白板、便签、细线。
- 透明小玻璃灵感瓶是“灵感捕捉/创意启动”的记忆物件，不是每张图的固定 pose。其他主题要换成对应动作和物件。
- 可选分析物件：放大镜、小秤、抽屉、折纸、蓝色反馈纸条。
- 人物轮廓保留手绘蜡笔感，不要精致商业插画感。

## 性格

- 温柔、明亮、亲切、靠谱。
- 有创作能量，但不过度可爱。
- 擅长把灵感变成内容，把复杂流程讲简单。
- 做判断时清醒克制，不热闹，不夸张。
- 像一个懂内容、懂工具、懂节奏的陪跑导师。

## 常见职责

让星禾承担核心动作：

- 灵感捕捉：举起、打开或倾倒装着星星的透明小玻璃瓶。
- 把星星灵感倒进内容成型装置。
- 把散乱灵感卡片排进内容日历。
- 用橙色线连接“选题 -> 生成 -> 发布 -> 复盘”。
- 把重复工作交给一个小自动化盒子。
- 指向蓝色反馈纸条，让复盘回流到下一轮选题。
- 在小白板前圈出容易断掉的运营节点。
- 把同一个灵感压成短帖、长文、视频脚本等不同内容卡。
- 用放大镜看一个问题结节。
- 用小秤称量“证据”和“感觉”。
- 把一团线拆成清晰路径。

## 人物缩放和可省略

技术架构图、流程图和知识卡片的第一目标是读者看懂结构。人物呈现按信息密度选择：

- `full-character`：人物完整出现，适合封面、情绪图、正文锚点图。
- `small-character`：人物占画面 5%-15%，适合在架构图/流程图里指向、接线、圈边界、贴标签。
- `partial-character`：只画手、半身、侧影或小小的标注动作，适合高密度卡片和横向流程。
- `no-character`：节点、流程或卡片信息已经很密，人物会干扰阅读；保留星禾风格的冷白/浅蓝灰底蜡笔、手写批注和色彩规则即可。

如果选择 `no-character`，不要声称画面里有星禾人物；它是“星禾风格结构图”。如果出现任何星禾人物、手部或半身，仍要用人物基准图约束形象。

## 动作必须跟内容绑定

不要把星禾固定成同一个姿势。先判断图片要表达的内容，再选择动作：

- 断点/卡住：星禾搬起堵住传送带的素材箱、接住掉落内容、把断开的橙色线重新接上。
- 最小闭环：星禾蹲在小机器旁插线、按按钮、把反馈纸条接回输入口。
- 分类/分发：星禾双手整理内容卡片，把卡片分别放进不同盒子或通道。
- 复用/一鱼多吃：星禾操作压印机、切分灵感纸、把同一份素材压成多种内容卡。
- 交接/承接：星禾在路径中间递交卡片、挂标签、把内容接到门把手或工具箱。
- 三类来源：星禾拉动三根线，把外部信号、内部资产、用户反馈收束到选题桶。
- 信任/证据：星禾俯身铺证据卡、搭小桥、用放大镜检查关键证据。
- 判断/优先级：星禾称量证据和感觉、圈出关键点、剪掉多余路线。

如果动作和图意无关，即使外形像星禾，也是不合格。

## 禁止

- 不要把星禾画成儿童卡通、二次元海报或精致商业插画。
- 不要大面积黄色背景；背景应为冷白或极浅蓝灰，可有轻量手绘分区。
- 不要过度可爱、眨眼、夸张表情包。
- 不要复杂服装、复杂配饰、华丽发光特效。
- 不要让星禾只站在角落里看；如果人物没有实际作用，就缩小成局部标注动作或不出现。
- 不要让星禾在所有图里都举瓶、挥手或保持同一姿势。
- 不要让星禾变成 PPT 讲师或课程海报人物。
- 不要让人物抢走结构表达；人物和概念必须共同成立。
- 不要把原图中的黄色满底照搬到正文配图里。

## 判断标准

正文图、封面、情绪图和漫画：如果去掉星禾，图的核心隐喻或运营流程还能完全成立，说明星禾太装饰了；要重写提示词，让星禾成为动作主体。

技术架构图、流程图和知识卡片：如果星禾会降低结构可读性，不要强行放完整人物；改用小人物、局部人物或无人物结构图。
