# 文字密度分级

本文件用于控制画面中文字数量。文字密度先由图型决定，再由发布场景和画面尺寸修正。

## 密度等级

| 等级 | 适用图型 | 建议文字量 |
|---|---|---|
| `very-low` | 平台封面、小红书首图 | 1 个标题 + 0-1 个短标签 |
| `low` | 情绪锚点图、正文锚点图 | 2-5 个短词或短标签 |
| `medium` | 解释图、流程图、单张知识卡 | 3-7 个节点，每个节点 2-10 字 |
| `medium-high` | 知识卡片组、技术架构图 | 1 个标题 + 3-5 个要点或 5-9 个节点 |
| `high` | 信息图海报、高密度总览图 | 3-6 个分区，最多 12 个短标签 |

## 图型上限

| 图型 | 默认密度 | 上限 |
|---|---|---|
| `platform-cover` | `very-low` | 标题 2-4 行，不加长副标题 |
| `emotion-anchor` | `low` | 5 个短标签 |
| `xinghe-article` | `low` | 5 个短标签 |
| `explanatory-diagram` | `medium` | 7 个节点 |
| `process-flow` | `medium` | 7 步；超过拆图或卡片组 |
| `technical-architecture` | `medium-high` | 9 个强关系节点 |
| `knowledge-card-single` | `medium` | 1 标题 + 3-5 要点 + 1 底部总结 |
| `knowledge-card-pack` | `medium` | 每张卡一个信息目标 |
| `comic-strip` | `low` | 每格最多 1 行短句 |
| `infographic-poster` | `high` | 6 个分区或 12 个短标签；超过拆卡 |

## 处理规则

- 标题、术语、数字必须来自原文或用户确认。
- 不把段落原文放进图里，先压缩成短标签。
- 如果文字超过上限，优先拆成多张卡，不缩小字号硬塞。
- 信息图可以高密度，但必须有分区、层级和阅读路径。
- 技术架构图可以节点多，但节点标签必须短，不写长解释。
- 小红书竖图优先大字可读，不能为了信息量牺牲移动端阅读。

## 输出字段

每个候选方向写：

```json
{
  "text_density_level": "medium",
  "text_budget": "1 标题 + 4 要点 + 1 底部总结",
  "text_overflow_plan": "如果要点超过 5 个，拆成 2 张卡"
}
```
