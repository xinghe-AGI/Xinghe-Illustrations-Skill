# 视觉 QA 评分样例

本文件用于校准 `visual-qa-rubric.md` 的评分口径。评分不是审美投票，而是判断图片是否能进入发布包、README 样张或正式内容生产。

## 样例 1：可发布的知识卡片

```text
文件：images/card-02-overview.png
路由：knowledge-card-single
总分：11 / 12
```

| 维度 | 分数 | 理由 |
|---|---:|---|
| 路由匹配 | 2 | 内容是 4 个相关知识点，放在同一卡片里有清晰关系 |
| 信息可读性 | 2 | 主标题、四个模块和底部总结路径明确 |
| 中文文字 | 2 | 标签来自原文，短而准，无随机文字 |
| 星禾参与方式 | 1 | 星禾在旁边整理卡片，动作有效但不是核心主体 |
| 构图和比例 | 2 | 4:3 横版适合文章正文和桌面阅读 |
| 风格边界 | 2 | 冷白浅蓝灰背景，手写笔记感稳定 |

结论：可发布。若作为 README 样张，可轻微增强人物动作，但不需要重生。

## 样例 2：必须返工的封面

```text
文件：images/xhs-cover-a.png
路由：platform-cover / xhs-cover
总分：7 / 12
```

| 维度 | 分数 | 理由 |
|---|---:|---|
| 路由匹配 | 2 | 是小红书封面，标题和人物结构基本对 |
| 信息可读性 | 1 | 标题层级能看懂，但关键词被装饰线遮挡 |
| 中文文字 | 0 | 标题出现错字，且模型额外生成无关小字 |
| 星禾参与方式 | 1 | 人物姿势偏摆拍，但没有遮挡标题 |
| 构图和比例 | 2 | 3:4 比例正确，安全边距基本可用 |
| 风格边界 | 1 | 装饰偏多，有一点商业海报感 |

结论：必须返工。封面标题错字是硬错误，不因总分接近 8 分而放行。返工策略：减少画面文字，只保留用户确认标题和 1 个关键词强调。

## 样例 3：结构正确但人物过大的架构图

```text
文件：images/architecture-layered-a.png
路由：technical-architecture
总分：8 / 12
```

| 维度 | 分数 | 理由 |
|---|---:|---|
| 路由匹配 | 2 | 分层架构、输入输出和支撑层都存在 |
| 信息可读性 | 1 | 主结构可读，但中心人物遮住两个节点 |
| 中文文字 | 2 | 节点短词清楚，术语来自原文 |
| 星禾参与方式 | 0 | 架构图里人物过大，抢走结构主体 |
| 构图和比例 | 1 | 16:9 合理，但中央区域拥挤 |
| 风格边界 | 2 | 视觉风格稳定，不像真实 UI |

结论：不建议发布，需返工。改 `character_presence` 为 `small-character` 或 `no-character`，把人物移到边角检查点，结构主体居中。

## 样例 4：全景信息图信息太满

```text
文件：images/panorama-system-map-a.png
路由：panoramic-infographic
总分：8 / 12
```

| 维度 | 分数 | 理由 |
|---|---:|---|
| 路由匹配 | 2 | 内容确实需要系统全貌和主路径 |
| 信息可读性 | 1 | 有阅读路径，但右侧栏和底部工具带过密 |
| 中文文字 | 1 | 大部分标签可读，但出现两处过长句 |
| 星禾参与方式 | 2 | 无人物，符合结构主导图规则 |
| 构图和比例 | 1 | 横版正确，但信息区超过建议上限 |
| 风格边界 | 1 | 有轻微 PPT 模板感 |

结论：内部参考可用，正式发布前应返工。保留 1 张全景图讲关系，把细节拆成 5 张知识卡片；右侧栏最多保留 3 个短模块。

## 样例 5：情绪图不够情绪化

```text
文件：images/emotion-anchor-low-mood.png
路由：emotion-anchor
总分：9 / 12
```

| 维度 | 分数 | 理由 |
|---|---:|---|
| 路由匹配 | 2 | 内容是卡住、焦虑和转折，适合情绪图 |
| 信息可读性 | 2 | 一眼能理解“卡住”的情境 |
| 中文文字 | 2 | 只有 3 个短标签，无错字 |
| 星禾参与方式 | 1 | 人物动作能表达卡住，但表情偏平，情绪不够明显 |
| 构图和比例 | 1 | 画面右侧略空 |
| 风格边界 | 1 | 背景略单调，缺少低密度批注 |

结论：可作为内部参考；如果用于发布，建议补强表情、姿态和环境压力，而不是增加文字。

## 快速判定口径

- 标题、关键术语、数字错了：直接返工。
- 人物风格漂移：直接返工，除非该图明确是 `no-character`。
- 结构图人物过大：通常扣到 0-1 分，并优先改小或删除人物。
- 全景图太满：不要缩小字号硬塞，拆成“全景图 + 卡片组”。
- 分数 10 分以上才适合进入 README 样张；8-9 分只能作为内部参考。
