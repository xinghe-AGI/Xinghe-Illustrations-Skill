# 视觉质量评分表

本文件用于真实生图后的质量评估，也用于决定是否返工。评分不是为了追求满分，而是帮助判断图片是否可以进入 README、文章发布包或正式内容生产。

## 使用方式

每张图按 6 个维度评分，每项 0-2 分，总分 12 分：

- `2 分`：达到要求，可以发布或作为样张。
- `1 分`：基本可用，但需要轻微返工或只适合内部参考。
- `0 分`：明显不合格，需要重生、拆图或改路由。

建议阈值：

- `10-12 分`：可发布 / 可进入 README 样张。
- `8-9 分`：可作为内部参考；正式发布前建议返工一次。
- `7 分及以下`：不建议发布，应按失败类型重写 prompt 或换路由。

## 评分维度

| 维度 | 2 分 | 1 分 | 0 分 |
|---|---|---|---|
| 路由匹配 | 图型和内容目标匹配，解释图、情绪图、架构图、流程图、漫画、卡片、信息图没有混用 | 图型大体正确，但有一点像另一类图 | 图型明显错用，例如架构内容做成普通插画 |
| 信息可读性 | 读者能快速看懂主路径、层级、关系或情绪转折 | 主体能看懂，但局部关系或阅读顺序不够清楚 | 信息散乱、过密、过空或没有清楚主线 |
| 中文文字 | 文字短、准、可读，关键术语基本来自原文或用户确认 | 有少量字形不稳、轻微错字或标签不够精确 | 错字多、文字过小、乱造词、出现无关文字 |
| 星禾参与方式 | 人物大小和动作服务信息；结构图中人物小、局部或不出现也合理 | 人物存在但动作略装饰，或结构图人物稍微抢主体 | 人物只是摆拍、遮挡信息、漂移严重或多人物失控 |
| 构图和比例 | 比例匹配用途；留白、分区、动线、卡片、节点和背景信息层稳定 | 比例可用，但部分区域偏空、偏挤或背景承载不足 | 比例不适合内容，重要信息被裁切、布局崩坏或大块空白 |
| 风格边界 | 保持冷白/浅蓝灰、蜡笔、手写笔记感；背景丰富但功能明确，配色不只蓝/橙/黑，不像 PPT/商业海报/儿童画 | 局部有模板感、颜色单调或装饰稍多，但总体还在边界内 | 明显变成 PPT、商业海报、儿童插画、真实 UI、第三方风格复刻，或背景小物与内容无关 |

## 必须返工的情况

出现以下任一情况，即使总分不低也要返工：

- 出现二维码、联系方式、真实密钥、真实 token、真实域名或无关品牌 logo。
- 复制了第三方参考图的具体人物、文字、水印、构图或品牌元素。
- 含人物任务没有保持星禾基准特征，或把星禾画成另一个 IP。
- 技术架构图、流程图、知识卡片因人物过大导致结构不可读。
- 封面标题、关键数字、术语出现明显错字。
- 信息图把文章段落硬塞成小字长文。

## 返工建议

按最低分维度选择返工方式：

- 路由匹配低：回到 `route-scoring.md`，重新选择图型。
- 信息可读性低：减少节点、增加分区、拆成卡片组或改成横版。
- 中文文字低：减少文字量，只保留标题、短标签和数字；必要时重生。
- 人物参与低：改 `character_presence`，让人物小一点、局部出现或不出现。
- 构图比例低：按 `prompt-template.md` 的平台尺寸预设重选比例。
- 风格边界低：重写 Visual DNA，强调冷白/浅蓝灰、蜡笔、手写笔记感、功能背景信息层、低饱和多色层级，禁止 PPT、商业海报和无关装饰。

## 记录格式

评估样张或正式输出时，可使用：

```text
文件：images/01-topic-a.png
路由：knowledge-card-single
总分：10 / 12
扣分：
- 中文文字：1 分，原因：一个标签略不稳定
- 构图比例：1 分，原因：右下角略空
结论：可用；正式发布前可轻微返工
```

评分样例见 `visual-qa-scoring-examples.md`。当用户问“这张图能不能用”“为什么要返工”“README 样张是否合格”时，先按本表评分，再参考样例校准口径。
