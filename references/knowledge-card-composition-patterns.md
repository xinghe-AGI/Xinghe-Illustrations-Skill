# 知识卡片构图骨架

本文件只记录知识卡片的信息结构和排版骨架，不参考任何外部图片的画风、色系、字体、图标、具体文字或科技 UI 装饰。

来源观察：`AI时代软件交付路径全景图_V1.0/01-08.png`。可借鉴的是“如何组织复杂信息”，不是视觉风格。

## 使用原则

- 先判断内容关系，再选构图：因果、路径、决策、分层、对比、汇聚、时间线和结果导向不能混用。
- 横版知识卡优先承载关系：`4:3` 用于普通文章笔记和方法论总结，`16:9` 用于路径、时间线、多模块关系和全景结构。
- 竖版只在小红书轮播、微信竖向图文、手机连续阅读或用户明确要求时优先。
- 一张卡只保留一个主结构。不要把路径图、决策树、三列对比和时间线全塞在同一张卡。
- 星禾在结构型知识卡中默认小、局部或不出现，只做指向、贴签、接线、圈边界；不要占据主结构。

## 横版构图模式

### 1. 左因右果路径卡

适合：解释“旧问题 -> 转折机制 -> 新结果”、从困境到方案、从低效到高效。

结构：

- 左侧：问题组或旧状态，通常 2-4 个并列小卡。
- 中间：转折节点、关键机制或核心判断。
- 右侧：新状态、收益、结果或可选方向，通常 2-3 个结果组。
- 主路径从左到右；中间节点要短、醒目。

不要：把右侧结果写成大段解释；不要让中间节点变成装饰。

### 2. 顶层原则 + 三列路径卡

适合：方法论总览、三种方案、三条路径、轻中重选择、能力分层。

结构：

- 顶部：一条横向总原则/底层逻辑/共同前提。
- 中部：三列主路径，每列有标题、适用场景、壁垒/风险、典型动作。
- 底部：组合策略、共同终点或行动建议。
- 三列之间保持平衡，每列最多 4-6 个短项。

不要：把三列做成纯列表；每列必须有不同定位。

### 3. 左定义右分层卡

适合：解释一个概念由哪些部分组成，以及这些部分如何对应价值层、场景层或案例层。

结构：

- 左侧：概念定义或组成部分，2-3 个大块。
- 右侧上部：价值分层、优先级或判断标准。
- 右侧下部：典型场景、应用清单或案例列表。
- 左右之间用 2-3 条连接线表达“组成 -> 价值/场景”的映射。

不要：右侧堆太多平级条目；优先分成“价值层”和“场景层”两块。

### 4. 上平台下模块卡

适合：工具平台、能力底座、工作流搭建、系统骨架。

结构：

- 顶部：平台/底座/核心能力条，放 2-4 个关键组件。
- 下方：分成 2-3 个功能模块区，分别说明优势、执行方式、注意点。
- 侧边可放一个“容易被低估/关键提醒”的小块。
- 从顶部到底部有清晰垂直分发线。

不要：把工具名当主角；主角应该是能力如何分发到模块。

### 5. 顶部原则分发卡

适合：原则、禁忌、正确做法、工具清单、行动指南。

结构：

- 顶部：3 个原则或总纲，横向排列。
- 中下部：原则向下分发到 2-3 个板块，例如“让路/正确做法/常用工具”。
- 每个板块内部使用短条目卡，形成清晰组块。
- 适合 `16:9`，也可压缩为 `4:3`。

不要：把原则和下方板块断开；要有分发关系。

### 6. 三输入汇聚结果卡

适合：多来源组合、三种能力合成一个方案、输入到结果、资源编排。

结构：

- 左侧：3 个纵向输入卡，每个卡有短标题和一句说明。
- 中间：汇聚线或合流节点。
- 右侧：一个大的结果卡，写清最终产出或核心结论。
- 输入卡可以用不同短标签区分，但不要复制外部图的色系。

不要：让右侧结果太小；汇聚结果必须是视觉重心。

### 7. 横向角色进化卡

适合：从 A 到 B 的身份变化、能力升级、职责迁移、阶段演进。

结构：

- 左侧：起点角色/旧状态。
- 中间：过渡角色/新定位。
- 右侧：不变能力、核心原则或目标状态。
- 使用连续箭头或路径线，表达“从 -> 到 -> 不变”。

不要：只写三个并列概念；必须体现变化方向。

## 竖版构图模式

### 8. 决策树路径卡

适合：怎么选、是否可行、三路选择、先问问题再分流。

结构：

- 顶部：入口问题或用户需求。
- 中部：一个核心决策节点，可以是菱形或醒目判断框。
- 下方：2-3 条分支路径，每条分支有条件和推荐路线。
- 底部：共同建议、MVP、下一步或收束结论。

适用比例：`3:4` 竖版最稳，也可在 `4:3` 中做横向决策树。

不要：分支条件含糊；每条路径都必须有“何时选它”的条件。

## 选择规则

| 内容关系 | 优先构图 |
|---|---|
| 旧问题到新结果 | 左因右果路径卡 |
| 三条路径、三种方案、三类选择 | 顶层原则 + 三列路径卡 |
| 概念组成与价值/场景映射 | 左定义右分层卡 |
| 工具底座、平台能力、模块装配 | 上平台下模块卡 |
| 原则、禁忌、正确做法、工具清单 | 顶部原则分发卡 |
| 多输入汇聚为一个方案 | 三输入汇聚结果卡 |
| 角色/能力阶段演进 | 横向角色进化卡 |
| 怎么选、是否可行、条件分流 | 决策树路径卡 |

## Prompt 字段

知识卡片候选方向必须补充：

- `composition_pattern`：从上表选择一个模式。
- `why_this_pattern`：说明该模式匹配的内容关系。
- `layout_flow`：读者视线如何移动，例如“左 -> 中 -> 右”或“上 -> 决策 -> 分支 -> 底部结论”。
- `primary_panel`：视觉重心是哪一块。
- `secondary_panels`：辅助信息放在哪些块。
- `character_presence`：`small-character` / `partial-character` / `no-character` 优先，只有封面卡才考虑 `full-character`。

## 禁忌

- 不复制参考图中的深色科技感、发光边框、图标、具体路径名、品牌、文字和色系。
- 不把这些模式做成 PPT 模板；星禾版仍然要保持冷白/浅蓝灰底、手绘分区、短标签和内容笔记感。
- 不强行把所有卡都做成同一种模式。卡片组中可以混用 2-3 种骨架，但每张卡只能有一个主骨架。
