# 知识卡片组叙事结构

本文件定义知识卡片组如何从散卡变成完整阅读路径。做 `knowledge-card-pack` 前先选一个叙事结构。

## 叙事结构

### 1. 问题 -> 原因 -> 方法 -> 案例 -> 总结

适合：方法论、项目复盘、工具介绍。

卡片顺序：

1. 问题卡：读者为什么会卡住。
2. 原因卡：问题背后的机制。
3. 方法卡：核心解决路径。
4. 案例卡：一个具体场景如何应用。
5. 总结卡：行动建议或收藏清单。

### 2. 误区 -> 真相 -> 做法 -> 检查清单 -> 收藏卡

适合：观点文章、避坑文、认知纠偏。

卡片顺序：

1. 误区卡：常见错误理解。
2. 真相卡：真正的判断。
3. 做法卡：如何落地。
4. 检查卡：发布前或执行前检查。
5. 收藏卡：一句话总结和复用场景。

### 3. 背景 -> 旧流程 -> 新流程 -> 工具链 -> 落地建议

适合：工作流、自动化、项目复盘。

卡片顺序：

1. 背景卡：为什么需要改变。
2. 旧流程卡：原本哪里低效。
3. 新流程卡：新的关键路径。
4. 工具链卡：各工具承担什么。
5. 建议卡：如何从小处开始。

### 4. 总览 -> 分层解释 -> 决策树 -> 流程图 -> 行动卡

适合：复杂方法论、高密度说明。

卡片顺序：

1. 总览卡：整套框架一眼看懂。
2. 分层卡：结构由哪些层组成。
3. 决策卡：什么情况下选哪条路。
4. 流程卡：执行顺序和回流点。
5. 行动卡：下一步怎么做。

### 5. 冲突 -> 转折 -> 重构 -> 稳定 -> 复用

适合：故事型文章、成长过程、工作区建设叙事。

卡片顺序：

1. 冲突卡：混乱、压力、卡点。
2. 转折卡：发现关键机制。
3. 重构卡：重新组织流程或规则。
4. 稳定卡：系统开始可靠运行。
5. 复用卡：沉淀成模板或规则。

## 选择规则

- 如果文章以“解决问题”为主，选结构 1。
- 如果文章以“纠正认知”为主，选结构 2。
- 如果文章以“流程升级”为主，选结构 3。
- 如果文章以“复杂框架解释”为主，选结构 4。
- 如果文章以“前后变化”为主，选结构 5。

## 输出字段

```json
{
  "card_pack_narrative": "背景 -> 旧流程 -> 新流程 -> 工具链 -> 落地建议",
  "card_count": 5,
  "card_roles": ["背景卡", "旧流程卡", "新流程卡", "工具链卡", "建议卡"]
}
```

## 质量要求

- 卡片组必须有阅读顺序，不是一组平行海报。
- 每张卡只解决一个阅读任务。
- 如果某张卡必须塞两个任务，优先拆卡。
- 第一张卡要让读者知道为什么要继续看。
- 最后一张卡要给行动、检查或复用建议。
