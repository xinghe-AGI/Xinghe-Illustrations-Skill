# 智能选点与图型选择

用于正文配图和文章锚点图的选点。是否走情绪图、解释图、多格漫画、知识卡片或信息图海报，由 `cognitive-anchor-routing.md` 和 `visual-formats.md` 决定；本文件不再维护全局视觉路由。

## 选点信号

优先画这些内容锚点：

| 信号 | 配图价值 | 星禾动作方向 |
|---|---|---|
| 抽象概念首次出现 | 高：帮助读者建立心智模型 | 星禾把抽象词放进瓶子、盒子、秤或白板 |
| 流程或步骤描述 | 高：动作比文字更清楚 | 星禾接线、搬卡片、按按钮、铺路径 |
| 比较、取舍或优先级 | 高：适合并排判断 | 星禾称量、圈选、移动卡片到不同区域 |
| 数据、证据或反馈 | 中高：能增强信任感 | 星禾捞出证据、铺证据卡、拉回反馈线 |
| 段落转场或阶段变化 | 中：给读者换气 | 星禾把内容从一个容器交接到另一个容器 |
| 情绪或故事高潮 | 中：增强共鸣 | 星禾扶住卡住的流程、修补断点、打开出口 |

谨慎处理这些：

- 只是定义、口号、背景铺垫，且没有动作。
- 已经很清楚的列表或常识判断。
- 需要大段文字才能解释的复杂系统：交给 `cognitive-anchor-routing.md` 判断是否拆成知识卡片、解释图、信息图海报或 diagram fallback。
- 只适合用正文讲的细节。

## 数量建议

| 内容长度 | 建议张数 |
|---|---|
| 短文或单条社媒内容 | 1-3 张正文锚点图 |
| 中等文章 | 2-4 张正文锚点图 |
| 长文或方法论文章 | 4-7 张正文锚点图 |
| 教程/SOP | 每个关键阶段 1 张，不机械按标题配 |

如果用户指定张数，以用户为准；否则优先少而准。

## 多候选方向

正文锚点图默认每个选点 2 个候选方向；重点图可给 3 个；快速任务、省成本任务或用户明确只要一个方向时给 1 个。候选字段和真实生成确认规则见 `visual-routing-and-candidates.md`。

每个候选方向必须包含：

- 候选编号：`A` / `B` / `C`
- 核心隐喻
- 星禾动作
- 构图
- 中文标注
- 推荐参考图
- 适用原因

如果当前段落更适合知识卡片、解释图、多格漫画、信息图海报或 diagram fallback，不要在本文件里硬转成正文配图；回到 `cognitive-anchor-routing.md` 重新路由。

## 图型分类

图型只用于帮助构思，不写进画面标题。

| 图型 | 适合内容 | 首选星禾动作 |
|---|---|---|
| `concept` | 抽象概念、心智模型 | 放进瓶子、贴到白板、围绕核心卡片整理 |
| `process` | 流程、步骤、自动化 | 接线、按按钮、搬开卡点、把卡放回轨道 |
| `comparison` | A/B 选择、策略取舍 | 称量、并排摆卡、用红笔圈出更优路径 |
| `data` | 数字、证据、反馈 | 捞取、筛选、贴证据、拉回反馈纸条 |
| `scene` | 用户场景、故事瞬间 | 亲手解决卡住的一刻，而不是旁观 |
| `metaphor` | 抽象类比、洞察表达 | 把概念变成低科技物件里的物理动作 |
| `handoff` | 承接、转化、交接 | 递卡、挂标签、连接门把手或路径 |
| `review-loop` | 复盘、迭代、回流 | 拉起回流线、夹住反馈纸条、送回输入端 |

## 输出模式

### 策略模式

当用户要“分析怎么配图 / 出配图方案 / shot list”时，输出每个配图选点，并在选点下给出 `candidates[]`：

- 放在哪段后
- 选点信号
- 图型
- 结构类型
- 推荐候选：默认标出 `A` 或最稳方向
- 候选方向 A/B/C：每个候选都写核心隐喻、星禾动作、构图、主要物件、建议中文标注、推荐参考图和适用原因

正文配图不要只给“方案一/方案二”的标题，要让候选在动作、构图或隐喻上真的不同。封面候选至少覆盖：标题强表达、人物动作强表达、留白/品牌感强表达。

### Prompt-only 模式

当用户要“只给提示词 / prompt-only / 不生成图片”时，只输出 shot list 和每个候选方向的完整 prompt，不调用生图工具。prompt-only JSON 中每个选点使用 `candidates[]`，每个 candidate 对应一个独立 prompt。

### 生成模式

当用户明确要“生成 / 做图 / 输出图片”时，先完成选点、视觉路由和候选方向判断。如果有多个候选，先确认要生成哪一个；如果用户明确要多个候选，逐个候选独立生成，文件名使用 `01-topic-a.png`、`01-topic-b.png`、`cover-a.png` 等，不覆盖旧图。

## QA 反问

每张图生成前先回答：

1. 这张图为什么放在这里，而不是随便配一张？
2. 它属于哪种图型？
3. 星禾正在亲手解决什么问题？
4. 如果删掉星禾，画面是否还成立？如果成立，说明星禾只是装饰，必须重写。
