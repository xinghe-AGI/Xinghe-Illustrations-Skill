# 风格 DNA

## 一句话

白底蜡笔线稿、大量留白、元气但克制、亲切但不幼稚、像白纸上的内容创作与运营笔记。比例由内容形态决定，可以是 3:4、4:3、1:1、2.35:1 或 16:9。

视觉目标：把灵感、内容、运营流程和技术结构画成一张轻盈、清爽、能一眼看懂的手绘解释图。正文图和封面让星禾参与核心动作；技术架构图、流程图和高密度知识卡片让结构成为主体，星禾可以很小、局部出现或不出现。

## 必须

- 根据内容形态选择比例：知识卡片和小红书优先 3:4，解释图和左右对比优先 4:3，单概念可用 1:1，公众号封面用 2.35:1，横版正文图才用 16:9。
- 纯白背景：不要米色、暖灰、纸张纹理、渐变、阴影、噪点、复古纸感。
- 蜡笔线稿：轻微颗粒、略带抖动、手绘感明显，不要机械矢量线。
- 主体线条以黑色、深蓝黑、蓝色蜡笔为主；星禾头发可用黑色或深蓝黑蜡笔堆叠。
- 大量留白：主体占画面约 40%-60%，至少 35% 空白，最好有一整块安静区域。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 一张图只讲一个核心动作、流程、状态、判断、隐喻或结构关系。
- 结构要自然表达，不要在图上写结构类型名称。
- 技术架构图、流程图、知识卡片可使用“小人物/局部人物/无人物”模式，不要为了露出完整人物牺牲结构可读性。

## 颜色

- 黑色/深蓝黑：星禾头发、主体线稿、主要物件、核心结构。
- 蓝色：衣褶、补充说明、AI/assistant/自动化状态、反馈、第二层解释。
- 橙色：主流程、路径、箭头、灵感到内容的移动关系、行动线。
- 红色：关键提醒、断点、风险、结果、结论圈注。

颜色要克制。红橙蓝用于短批注、箭头和少数视觉锚点；即使是知识卡片或信息图海报，也要保持白底、低饱和和手绘笔记感。

## 常用视觉母题

- 灵感瓶和星星：表达捕捉灵感、创意启动、个人 IP 记忆点。
- 内容卡片：表达短帖、长文、视频脚本、发布素材。
- 小白板和橙色线：表达轻量运营流程。
- 自动化盒子和蓝色反馈纸条：表达 AI 工作流、复盘回流、重复工作被接住。
- 放大镜、小秤、红色圈注：表达清醒判断、证据、关键结论。

每张图只选 1-2 个母题，不要堆满。

## 绝对不要

- 不要商业插画。
- 不要空泛、冷硬、模板化的 PPT 信息图。
- 不要冷硬模板化的 PPT 架构图；但可以做白底蜡笔风格的技术架构图、流程图、结构图和节点关系图。
- 不要课程课件。
- 不要可爱卡通海报。
- 不要儿童插画。
- 不要把复杂架构图硬塞成一张大人物 IP 图；必要时让人物变小、局部出现、不出现，或拆图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要复杂背景、渐变、阴影、纹理。
- 不要大面积黄色背景。
- 不要把每个节点都解释清楚；复杂内容要拆成多张图、卡片组或信息图海报。
- 不要左上角写“运营流程 / Workflow / 系统架构图 / 自动化 SOP / 研究框架 / 路线图”等类型标题。

## 审美方向

要明亮、亲切、清爽、可靠、有行动感，同时保留一点清醒和克制。

不要幼稚、甜腻、复杂、死板、营销海报感。
