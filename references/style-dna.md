# 风格 DNA

## 一句话

冷白/浅蓝灰底蜡笔线稿、有结构的呼吸感留白、元气但克制、亲切但不幼稚、像一张清爽的内容创作与运营知识卡。比例由内容形态决定，可以是 3:4、4:3、1:1、2.35:1 或 16:9。

视觉目标：把灵感、内容、运营流程和技术结构画成一张轻盈、清爽、能一眼看懂的手绘解释图。正文图和封面让星禾参与核心动作；技术架构图、流程图和高密度知识卡片让结构成为主体，星禾可以很小、局部出现或不出现。

## 必须

- 根据内容形态选择比例：小红书/微信竖图/手机轮播优先 3:4，文章笔记和方法论知识卡优先 4:3，横向流程/时间线/多模块关系优先 16:9，单概念可用 1:1，公众号封面用 2.35:1。
- 冷白或极浅蓝灰背景：允许浅蓝灰分区、细点阵、手绘虚线、浅色模块底和轻量批注承载信息；不要米色、暖灰、纸张纹理、渐变、阴影、噪点、复古纸感。
- 蜡笔线稿：轻微颗粒、略带抖动、手绘感明显，不要机械矢量线。
- 主体线条以黑色、深蓝黑、蓝色蜡笔为主；星禾头发可用黑色或深蓝黑蜡笔堆叠。
- 呼吸感留白：画面不能挤满，但解释图、流程图、知识卡片不要空到信息漂浮。普通正文图保留 30%-40% 安静区域；结构图和卡片可降到 20%-35%，用浅色分区承接内容。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 一张图只讲一个核心动作、流程、状态、判断、隐喻或结构关系。
- 结构要自然表达，不要在图上写结构类型名称。
- 技术架构图、流程图、知识卡片可使用“小人物/局部人物/无人物”模式，不要为了露出完整人物牺牲结构可读性。

## 颜色

- 黑色/深蓝黑：星禾头发、主体线稿、主要物件、核心结构。
- 蓝色/蓝灰：衣褶、背景分区、补充说明、AI/assistant/自动化状态、反馈、第二层解释。
- 橙色：可用于标题字、关键词、短标签、主流程、路径、箭头和行动线；它是文字层级的一种可选颜色，不只是一种强调线。
- 红色：关键提醒、断点、风险、结果、结论圈注。

颜色要克制但不单调。冷色底负责清爽和结构，黑色/深蓝黑负责基础可读，橙色可以直接承担部分标题、关键词或短标签文字，蓝色/蓝灰负责分区和辅助说明。即使是知识卡片或信息图海报，也要保持低饱和、手绘笔记感和清晰信息层级；不要把所有文字都统一成橙色。

## 常用视觉母题

- 灵感瓶和星星：表达捕捉灵感、创意启动、个人 IP 记忆点。
- 内容卡片：表达短帖、长文、视频脚本、发布素材。
- 小白板和橙色线：表达轻量运营流程。
- 自动化盒子和蓝色反馈纸条：表达 AI 工作流、复盘回流、重复工作被接住。
- 放大镜、小秤、红色圈注：表达清醒判断、证据、关键结论。

每张图只选 1-2 个母题，不要堆满。

## 绝对不要

- 不要商业插画。
- 不要空泛、冷硬、模板化的 PPT 信息图。
- 不要冷硬模板化的 PPT 架构图；但可以做冷白/浅蓝灰底蜡笔风格的技术架构图、流程图、结构图和节点关系图。
- 不要课程课件。
- 不要可爱卡通海报。
- 不要儿童插画。
- 不要把复杂架构图硬塞成一张大人物 IP 图；必要时让人物变小、局部出现、不出现，或拆图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要复杂背景、渐变、阴影、纹理；但允许极浅冷色结构底、点阵、分隔线和手绘模块。
- 不要大面积黄色背景。
- 不要把每个节点都解释清楚；复杂内容要拆成多张图、卡片组或信息图海报。
- 不要左上角写“运营流程 / Workflow / 系统架构图 / 自动化 SOP / 研究框架 / 路线图”等类型标题。

## 审美方向

要明亮、亲切、清爽、可靠、有行动感，同时保留一点冷静、秩序和克制。解释图要比普通插图更有信息承载面，不能只有空白背景和几个漂浮元素。

不要幼稚、甜腻、复杂、死板、营销海报感。
