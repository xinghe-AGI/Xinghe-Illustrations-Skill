# 中文文字渲染规则

生图里的中文文字是高风险区。默认少写、写准、写短。

## 文字来源

允许写入画面的文字：

- 用户明确给出的封面标题。
- 原文里的关键术语、步骤名、数字、对象名。
- 用户确认过的改写标题或短标签。

不要写：

- 未核验数字。
- 未出现过的商业承诺。
- 二维码、联系方式、价格、购买引导。
- 色值、字体名、提示词参数或模型术语。
- 大段正文、完整代码、密集表格。

## 字数密度

| 图形 | 建议文字 |
|---|---|
| 情绪锚点图 | 1 个短标题或 2-5 个短标签 |
| 解释图 | 3-7 个节点短词，每个 2-8 字 |
| 多格漫画 | 每格 0-1 句短对白或短标签 |
| 知识卡片 | 1 个主标题 + 3-5 个要点 + 可选底部总结 |
| 信息图海报 | 分区标题短，正文尽量用关键词 |
| 公众号封面 | 1-2 行短标题 |
| 小红书封面 | 2-4 行大字标题 |

如果文字超过限制，优先拆成多张卡或多张图。

## 标题规则

- 标题必须逐字来自用户确认版本。
- 需要英文时保留原文大小写，例如 Codex、AI、API、Workflow。
- 标题不要贴边，移动端保留安全边距。
- 只强调 1 个关键词，使用下划线、圈注或轻色块。

## 错字处理

- 错字少：可局部编辑或重生。
- 错字多：减少画面文字后重生。
- 不用代码在图片上涂改错字；这会破坏手绘质感和可信度。

## 代码、表格、UI

- 代码只能画成“代码卡片”或“命令纸条”的示意，不写长代码。
- 表格只能画成简化卡片或对比栏，不画密集真实表格。
- UI 只能抽象成低科技窗口或小白板，不生成真实 App 截图。

## 检查问题

交付前检查：

1. 画面文字是否都能在原文或用户标题中找到来源？
2. 手机端能否一眼读清？
3. 有没有错别字、乱码、随机英文或多余符号？
4. 是否出现了联系方式、二维码、价格或不该有的承诺？
5. 如果去掉一半文字，图是否更清楚？如果是，就删减。
