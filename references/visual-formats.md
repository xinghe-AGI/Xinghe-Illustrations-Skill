# 视觉形态规则

本文件定义星禾统一 skill 支持的主要视觉形态。选择形态时先看内容任务，再看平台尺寸。

## 情绪锚点图

适合：

- 误区与真相、理想与现实、压力与解法、卡住与打通。
- 读者需要先共鸣，再理解方法。
- 作为多格漫画、封面或正文图的辅助路由，用来定义情绪阶段、反差和动作强度。

情绪谱系：

- `低落 / 不开心`：肩膀下沉、眼神向下、手里拿着没完成的卡片，适合失败、卡住、没反馈。
- `慌张`：眼睛睁大、身体前倾、接住快掉落的任务卡，适合任务爆量、截止时间、突然出错。
- `烦躁`：眉头皱起、嘴角压低、拉扯打结线团或反复改稿，适合返工、反复沟通、流程打结。
- `迷茫 / 困惑`：停在白板前、手拿问号卡、看着多条分叉路径，适合方向不清、选择困难。
- `专注 / 紧绷`：抿嘴、眼神集中、正在接线或检查关键节点，适合高压执行、上线前检查。
- `释然 / 开心`：松一口气、把整理好的卡片放入盒子，适合问题解决、流程跑通、复盘完成。

构图：

- 星禾可以较大，占画面 35%-60%。
- 用一个强动作承载情绪：扶住快倒的资料堆、拉回断线、把混乱卡片装进盒子、从压力堆里开出口。
- 背景干净，旁边只放 2-5 个短标签。

禁忌：

- 不做纯表情包。
- 不默认微笑；除非语义是释然、开心、完成或鼓励。
- 不把情绪画成夸张哭喊或营销鸡血。
- 不让情绪盖过信息目标；情绪只是入口，最后必须回到文章观点。

## 解释图

适合：

- 抽象概念、轻量流程、因果链、机制、三步法、对比关系。

构图：

- 星禾可以小一些，占 15%-30%，作为结构里的操作者。
- 允许有框、箭头、分区、编号，但必须像白纸上的手绘解释，不像模板 PPT。
- 每个节点只放短词，不放完整段落。
- 如果结构超过 8 个强关系节点，建议拆图或 diagram fallback。

禁忌：

- 不做冷硬 PPT 架构图；如果内容本身就是系统架构，改走 `technical-architecture`。
- 不把每个节点都解释清楚；复杂度过高时拆分。

## 技术架构图

适合：

- 系统组件、服务边界、数据流、模块依赖、部署关系、权限边界。
- 技术说明、产品架构、Agent 工作流、自动化系统、数据链路。

构图：

- 结构是主角，人物不是主角。
- 默认 `4:3` 或 `16:9`，节点多时横向更稳。
- 先分层：入口层、处理层、存储层、外部服务、反馈层。
- 星禾默认使用 `small-character`、`partial-character` 或 `no-character`：可以只在角落拿笔圈边界、只露手贴标签，或完全不出现。
- 主路径用橙色，反馈/日志/状态用蓝色，风险/权限/失败点用红色。

禁忌：

- 不把架构图画成“大人物 + 几个随意盒子”。
- 不为了 IP 露出压缩节点、压小文字或破坏关系。
- 不画真实 UI、真实 logo、真实 token、真实域名或密钥。

## 流程图

适合：

- SOP、自动化流程、审批流、状态流、内容生产流程、任务流。
- 需要看清先后顺序、分支、回流和人工确认点的内容。

构图：

- 3-7 步可做单张；超过 7 步优先拆成知识卡片组或分段流程图。
- 主流程从左到右或从上到下；回流线用蓝色，风险点用红色。
- 星禾默认小或局部：拿笔标注、贴“检查/回流/确认”标签、在关键节点接线。
- 如果人物会遮挡流程，用 `no-character`。

禁忌：

- 不做课程页和模板 PPT。
- 不把所有步骤塞进一张小红书卡片。
- 不让星禾站在流程旁边摆 pose。

## 多格漫画

适合：

- 因果、累积、转折、误会解除、从混乱到清楚、从手工到自动化。
- 有明显情绪阶段：期待、卡住、崩溃、发现、行动、松一口气、复盘。

构图：

- 默认 2-4 格。
- 每一格必须推进一个动作或判断。
- 格与格之间保持同一视觉世界：同一工作台、同一任务线或同一问题。
- 星禾可以在多格里重复出现，但要保持人物一致，动作随剧情变化。
- 如果漫画和 `emotion-anchor` 组合，每格都要写清情绪功能：`误区期待`、`现实受阻`、`星禾介入`、`秩序建立`、`结果确认`。
- 情绪表达服务转折，不服务夸张表情。可以画压力、困惑、专注、释然，不画失控哭喊或鸡血胜利。

判断：

- 如果任意一格删掉后意思不变，就降级为单张解释图或知识卡片。
- 如果只是把一个列表拆成四格，不要用漫画。

## 知识卡片组

适合：

- 长文总结、小红书轮播、公众号小绿书贴图、方法论、项目复盘、教程和工具介绍。
- 路径选择、决策分流、能力分层、工具组合、角色进化、输入汇聚和方法论全景。

默认结构：

1. 封面 / 承诺卡
2. 总览卡
3. 核心观点卡
4. 对比 / 流程 / 清单卡
5. 总结 / 收藏卡

长文默认 5-9 张；短内容可减少。每张卡只承担一个信息目标。

知识点组织：

- 允许多个相关知识点放在同一卡片里，但必须同属一种关系：并列、因果、流程、分层、输入汇聚、决策分流、角色进化或左右对比。
- 如果知识点只是堆在一起，没有清楚关系，拆成多张卡。
- 如果同一卡片包含多个知识点，必须有一个主问题或主结论把它们连起来。
- 关系型卡片优先使用 `knowledge-card-composition-patterns.md` 的构图骨架，不要做成散乱便利贴。

版式：

- 先判断平台和信息结构，不默认竖版。
- `3:4` 只在小红书轮播、微信竖向图文、手机端连续阅读或用户明确要求竖版时优先。
- `4:3` 用于普通文章笔记、方法论总结、轻流程、对比卡、清单卡和阅读型知识卡。
- `16:9` 用于横向流程、时间线、多模块对照、桌面阅读、公众号正文横图型知识卡。
- `1:1` 用于单概念、摘录、九宫格或方形社媒图。
- 构图骨架从 `knowledge-card-composition-patterns.md` 选择，不要只套“上标题 + 三点清单”。
- 一张卡最多 1 个主标题、3-5 个要点、1 个底部总结条。
- 星禾可以是引导者、整理者、对比者、标记者，不要只是贴纸。
- 人物默认小一些：总览卡、流程卡、对比卡优先 `small-character` 或 `partial-character`；术语卡、清单卡、架构卡允许 `no-character`。

禁忌：

- 不把文章段落原样塞进卡片。
- 不用缩小字号硬塞。
- 不把卡片做成“普通插画 + 几个标签”。
- 不把无关系知识点塞进同一张卡；可以密，但必须有关系。

## 单张知识卡片

适合：

- 单个观点、单个流程、单个对比、单个清单、单个提醒。

常用结构：

- 标题区 + 星禾动作 + 3 点清单。
- 左右对比 + 星禾在中间称量或移动卡片。
- 顶部结论 + 中部流程 + 底部提醒。
- 左因右果路径卡：左侧旧问题，中间转折，右侧新结果。
- 顶层原则 + 三列路径卡：顶部共同前提，中部三种方案，底部组合建议。
- 决策树路径卡：顶部问题，中部判断节点，下方分支，底部收束结论。
- 三输入汇聚结果卡：左侧多输入，中间汇聚线，右侧结果卡。
- 无人物结构卡：顶部结论 + 中部 3-5 个模块 + 底部提醒，只保留星禾风格冷白/浅蓝灰底蜡笔和手写批注。

比例：

- 小红书、微信竖图、手机轮播优先 `3:4`。
- 文章笔记、方法论总结、对比/流程/结构卡优先 `4:3`。
- 横向流程、时间线、多模块关系和公众号正文横图优先 `16:9`。
- 单概念或方形社媒图可用 `1:1`。

## 信息图海报

适合：

- 整篇流程图、全局地图、矩阵、方法全貌、年度/项目复盘总览。

规则：

- 默认最多 1 张。
- 先锁定结构，再做布局。
- 可以使用分区、路线、编号、卡片堆、总览地图。
- 星禾可以小而关键：拿笔、指路、整理地图，而不是抢走信息主体。
- 信息图允许承载比普通正文图更多的文章信息，信息密度可以是 `high`。
- 高密度信息图必须使用 3-6 个清晰分区，每个分区只放短标题、关键词、数字或极短句。
- 信息图适合“全景、矩阵、路径、地图、框架、阶段、角色关系”，不适合复刻全文段落。
- 如果一张信息图需要超过 6 个分区或 12 个短标签，优先拆成知识卡片组。
- 信息图可以与 `knowledge-card-pack` 组合：海报负责全局地图，卡片负责逐点解释。

禁忌：

- 不把全文段落硬塞进一张海报；可以承载多信息，但必须短标签化和分层。
- 不做密集课程页、PPT 页或商业长图。

## 平台封面

公众号封面看“缩略图点击”，小红书封面看“竖屏停留”。封面任务读取平台封面专门 references。

封面可以有大字标题，但标题必须短、准、可读。星禾动作必须承托标题判断。
